# FISLAB-II Workspace Instructions

AI agents working on this codebase should understand the following architecture, conventions, and workflows.

---

## ­¤ōŗ Project Overview

**FISLAB-II** is a single-page laboratory management portal for ITS Physics Department (Laboratorium Fisika Dasar 2). It manages practicum scheduling, module delivery, grading, and student/staff administration.

- **Technology Stack**: HTML5, CSS3, JavaScript (ES6+), Firebase Realtime Database
- **Architecture**: Multi-Page Application (MPA) with modular JavaScript
- **Users**: Praktikan (students), Asisten Laboratorium (TAs), Admin (senior TAs)
- **No build bundler**: All code is vanilla JS loaded via CDN or inline

---

## ­¤ÅŚ’ĖÅ Architecture Patterns

### Authentication & Session Management

- **Entry Point**: `index.html` (login page)
- **Session Storage**: LocalStorage keys:
  - `fislab-nrp`: User NRP (unique ID)
  - `fislab-role`: User role (Praktikan/Asisten Laboratorium)
  - `fislab-admin`: Boolean flag for admin privileges
  - `fislab-theme`: Dark mode preference (light/dark)

- **Auth Guard Location**: `sidebar.js`
  - Executed on every page load via `DOMContentLoaded`
  - Redirects unauthenticated users to `index.html`
  - Redirects authenticated users away from `index.html` to `home.html`
  - Syncs across browser tabs using `storage` event

- **Credentials**: Validated against Firebase Realtime Database (`/users/{nrp}`)
  - Client-side validation (NRP + plaintext password check)
  - No email/OAuth integration yet

### Role-Based Access Control (RBAC)

Three roles with cascading permissions:

| Role | Pages | Features |
|------|-------|----------|
| Praktikan | home, profile, modul, nilai, jadwal, kontak | View own grades, schedules, modules, contact TAs |
| Asisten Laboratorium | home, profile, modul, jadwal, penjadwal, penilaian, kontak | All praktikan + grade, schedule management |
| Admin | home, profile, admin + all asisten pages | All asisten + export data, user management, analytics |

**Implementation**: `sidebar.js` hides/shows nav items based on `fislab-role` and `fislab-admin` flags.

### Real-Time Data Sync

- **Database**: Firebase Realtime Database (JSON tree structure)
- **Pattern**: `.once('value')` for one-time reads, `.on('value')` for live listeners
- **Nodes**:
  - `/users/{nrp}` ŌĆö User profiles, grades, credentials
  - `/kelompok/{kelompokId}` ŌĆö Group schedules and assignments
  - `/judulPraktikum/{modulId}` ŌĆö Module metadata
  - `/modul/{modulId}` ŌĆö Additional module content (optional)

### Module & LaTeX Conversion Pipeline

1. **Source Files**: LaTeX documents in `modul/MP/` (Modern Physics) and `modul/W/` (Waves)
2. **Conversion Script**: `scripts/convert-tex.js`
   - Uses Pandoc: `pandoc -f latex -t html --mathjax -o output.html input.tex`
   - Post-processes HTML: fixes image paths, removes injected MathJax scripts
   - Outputs to `public/modul-html/`
3. **Module Mapping**:
   - **MP1-MP5**: Modern Physics (Planck, Milikan, Hysteresis, Radioactivity, Franck-Hertz)
   - **W1-W5**: Waves (Diffraction, Polarization, Spectroscopy, Mechanical Waves, Newton Rings)
   - **Data**: Data Processing (separate module)

## ­¤ōü Directory Structure Logic

### Root Level HTML Pages

Each page is a self-contained MPA route with paired JS module:

- **Login**: `index.html` + `index.js`
  - Loading screen, avatar animation (Epstein Files/Eins*.png)
  - Firebase credential validation
  - Session initialization
  
- **Dashboard**: `home.html` + `home.js`
  - Chart.js visualizations (role-dependent)
  - Quick stats and module shortcuts
  
- **Profiles & Settings**: `profile.html` + `profile.js`
  - Edit user info, upload avatar (Base64 ŌåÆ Firebase)
  - Change password
  
- **Module Viewer**: `modul.html` + `view-modul.html` + `view-modul.js`
  - Grid of 11 modules (MP1-5, W1-5, Data)
  - LaTeX-rendered HTML viewer
  - MathJax for equations
  
- **Grading** (TA-only):
  - `penilaian.html` + `penilaian.js` ŌĆö Input 9 grading aspects + plagiarism penalty
  - `nilai.html` + `nilai.js` ŌĆö Student view of grades
  
- **Scheduling**:
  - `jadwal.html` + `jadwal.js` ŌĆö View schedules (list/calendar)
  - `penjadwal.html` + `penjadwal.js` ŌĆö TA: schedule practicum sessions
  
- **Admin**: `admin.html` + `admin.js`
  - User management, data export, analytics
  
- **Contacts**: `kontak.html` + `kontak.js`
  - TA directory with WhatsApp links

### Shared Components

- **`sidebar.js`**: Global module (loaded on every page)
  - Auth guards, navigation, dark mode, logout
  - Manages hamburger menu, overlay interactions
  
- **`style.css`**: Global styles
  - CSS custom properties for theming (--primary-gold, --card-bg, etc.)
  - Dark mode class on `body.dark`
  - Responsive grid and component library
  
- **`firebase-config.js`**: Centralized Firebase initialization
  - Attaches config to `window.__FISLAB_FIREBASE_CONFIG__`
  - Initializes Firebase SDK if loaded

### Build & Content Directories

- **`modul/`**: LaTeX source files
  - `MP/` ŌĆö Modern Physics modules (5 .tex files)
  - `W/` ŌĆö Wave modules (5 .tex files)
  - `Gambar/` ŌĆö Image assets (referenced by LaTeX)
  - `Pengolahan_Data.tex` ŌĆö Data processing module
  
- **`public/modul-html/`**: Generated HTML (output from Pandoc)
  - Created by `scripts/convert-tex.js`
  - 11 HTML files (MP1-5, W1-5, Data)
  - Images referenced relative to Gambar folder
  
- **`scripts/`**: Build and utility scripts
  - `convert-tex.js` ŌĆö Pandoc wrapper with post-processing
  
- **`loader/`** and **`Epstein Files/`**: Assets
  - `Epstein Files/Eins1.png` through `Eins12.png` ŌĆö Avatar animation frames
  - `loader/` ŌĆö Loading indicator assets
  - `favicon.svg` ŌĆö Site icon

- **File**: `supabase-config.js`

## ­¤ōŖ Grading System

### Grading Aspects (9 aspects + plagiarism penalty)

| Aspect | Key | Weight | Notes |
|--------|-----|--------|-------|
| Pre-lab | pre-lab | 10% | Preparation |
| In-lab | in-lab | 10% | Performance |
| Abstrak | abstrak | 5% | Report: abstract |
| Pendahuluan | pendahuluan | 10% | Report: introduction |
| Metodologi | metodologi | 5% | Report: methodology |
| Analisis Data & Perhitungan | analisis data dan perhitungan | 20% | Report: data analysis |
| Pembahasan | pembahasan | 25% | Report: discussion |
| Kesimpulan | kesimpulan | 10% | Report: conclusion |
| Format | format | 5% | Report: formatting |
| Plagiarism & AI Detection | plagiasi, dan tidak bisa mengurai AI | Penalty | Percentage deduction |

### Grading Formula

```javascript
totalAspek = ╬Ż(nilai_aspek ├Ś bobot_aspek / 100)
nilaiAkhir = totalAspek ├Ś (1 - plagiarism_penalty)
// plagiarism_penalty is plagiarism_percentage / 100
```

### Grade Status States

- **Belum Dinilai**: No grades entered (filledCount === 0)
- **Sedang Dinilai**: Partial grades (0 < filledCount < 9)
- **Sudah Dinilai**: All 9 aspects graded (filledCount === 9)

## ­¤öĆ Common Page Flows

### Login Flow
1. User ŌåÆ `index.html`
2. Enter NRP + password (or guest mode)
3. JS validates against `/users/{nrp}` in Firebase
4. On success: store `fislab-nrp`, `fislab-role`, `fislab-admin` in localStorage
5. Redirect to `home.html`
6. `sidebar.js` on home.html syncs navigation based on role

### Grading Workflow (TA)
1. Navigate to `penilaian.html`
2. Select kelompok ŌåÆ loads praktikan list from `/kelompok/{kelompokId}`
3. Select praktikan ŌåÆ loads existing grades from `/users/{nrp}/nilai/{modulId}`
4. Input 9 aspect values + plagiarism percentage
5. Submit ŌåÆ writes to `/users/{nrp}/nilai/{modulId}` with `_updatedAt` timestamp
6. Praktikan sees update in `nilai.html` (real-time via Firebase listener)

### Module Conversion Workflow
1. Edit LaTeX source in `modul/MP/*.tex` or `modul/W/*.tex`
2. Run `node scripts/convert-tex.js`
3. Pandoc converts each .tex ŌåÆ corresponding .html in `public/modul-html/`
4. Post-processor adjusts image paths to reference `modul/Gambar/`
5. User opens `modul.html`, clicks module ŌåÆ loads from `public/modul-html/{modulId}.html`
6. MathJax renders equations, images display inline

## ­¤Ä© UI/UX Conventions

### Dark Mode

- **CSS Class**: `body.dark` applied when `localStorage.getItem('fislab-theme') === 'dark'`
- **CSS Variables**: Switched in `:root` and `body.dark` selectors
  - `--primary-gold`: Main accent color (#CFA348)
  - `--bg-gradient`, `--card-bg`, `--text-color`, etc.
- **Toggle**: `#mode-toggle` button (sun/moon icons)

### Sidebar Navigation

- **Classes**: `sidebar-open` on body when sidebar visible
- **Hamburger Button**: `#hamburger` toggles sidebar
- **Overlay**: `#sidebar-overlay` closes sidebar on click
- **Responsive**: Hamburger hidden on desktop (CSS media query)
- **Keyboard**: Escape key closes sidebar

### Loading Screen

- **Trigger**: `showLoadingScreen()` in `index.js` during login
- **Avatar Animation**: 12-frame animation cycle (Epstein Files/Eins1-12.png)
- **Progress Bar**: Simulated progress to 90% over 4.5s, then completes
- **Auto-finish**: After 6 seconds (customizable)

## ­¤ōØ Code Conventions

### Naming Patterns

- **HTML IDs**: Kebab-case with prefixes: `#nav-modul`, `#login-form`, `#progress-bar`
- **CSS Classes**: Kebab-case: `sidebar-open`, `card-component`, `dark`
- **JS Constants**: UPPER_SNAKE_CASE: `AVATAR_FRAMES`, `MP_MAP`, `W_MAP`
- **User NRP**: 10-digit string (ITS format: `5026221234`)
- **Module IDs**: `MP1ŌĆōMP5` (Modern Physics), `W1ŌĆōW5` (Waves), `Data`

### File Organization

- **One `.html` per page** (structure + minimal inline scripts for CDN loading)
- **One `.js` per page module** (DOM logic, Firebase listeners, event handlers)
- **`sidebar.js`**: Global module loaded on all pages
- **`style.css`**: Single global stylesheet (CSS variables for theming)

### Firebase Patterns

```javascript
// Read once
firebase.database().ref('path').once('value').then(snapshot => {
  const data = snapshot.val();
  // Use data
});

// Listen for changes
firebase.database().ref('path').on('value', snapshot => {
  const data = snapshot.val();
  // Update UI
});

// Write
firebase.database().ref('path').set(data);

// Update (merge)
firebase.database().ref('path').update({ field: value });
```

## ­¤øĀ’ĖÅ Maintenance & Extending

### Adding a New Grading Aspect

1. Add key to grading system (e.g., "new_aspect")
2. Update bobot distribution (sum must equal 100%)
3. In `penilaian.html`: Add input field for aspect
4. In `penilaian.js`: Include in form submission to Firebase
5. In `nilai.js`: Add to grade breakdown display

### Adding a New Module

1. Create LaTeX file: `modul/MP/New_Module.tex` or `modul/W/New_Module.tex`
2. Edit `scripts/convert-tex.js`: Add mapping (e.g., `MP6: 'New_Module.tex'`)
3. Run `node scripts/convert-tex.js` to generate HTML
4. Add to `judulPraktikum` node in Firebase Console
5. Module auto-appears in `modul.html` grid

### Deploying to Production

- Recommended platforms: Firebase Hosting, GitHub Pages, Netlify, Vercel
- Ensure `firebase-config.js` has production credentials
- Run `node scripts/convert-tex.js` before deploying
- Test role-based pages thoroughly
